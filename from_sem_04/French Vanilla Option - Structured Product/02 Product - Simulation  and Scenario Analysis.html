def g_price_path(s0=s0,K=K,r=r,q=q,vol=vol,T=T,N=N,M=M,dt=dt):
    """Generates N values of stock price from time 0 to T.
    Stock price follow geometric brownian motion."""
    s = np.array([s0]) 
    temp1 = ((r - q) - (0.5 * vol*vol))*dt
    temp2 = vol * np.sqrt(dt)
    for step in range(N):
        next_val = s[step] * np.exp(temp1 + (temp2 * np.random.randn()))
        s = np.concatenate([s,[next_val]])
    return s

The function above, $g\_price\_path$, will make an entire path from time zero to time $T$ with $N$ steps. This information will be stored in a Numpy 1d-array. We can now call this function $M$ times, for the number of paths we'd like to create. Each path will be "appended" into a 2d-array (matrix).

def multi_path(s0=s0,K=K,r=r,q=q,vol=vol,T=T,N=N,M=M,dt=dt):
    build_multi = g_price_path(s0=s0,K=K,r=r,q=q,vol=vol,T=T,N=N,M=M,dt=dt)
    for sims in range(M-1):
            new_path = g_price_path(s0=s0,K=K,r=r,q=q,vol=vol,T=T,N=N,M=M,dt=dt)
            build_multi = np.vstack((build_multi, new_path))
    return build_multi

II. Payoffs

With our matrix of paths created, we can now calculate payoffs in each case, and finally the price of the product.

We have three things to look for:

  • stock price outside of our barriers at time $a$
  • stock price breaching the barriers between time $a$ to $b$
  • stock price final value at time $T$

We can generate a payoff array by checking the various cases, and setting a payoff value accordingly.

In [20]:
def payoff_calc(sp_matrix,check_dict):
    h_list = []
    # Garrison entry and exit times
    a,b = check_dict['a'], check_dict['b']
    # Full time to maturity
    Tm = len(sp_matrix[0])-1
    # High and Low price barriers
    H1,H2 = check_dict['H1'], check_dict['H2']
    
    def garrison_breach(some_path):
        """ find the payoff in the case the product was converted due to 
        exceeding any barrier, at all, between times a and b."""
        
        g_payoff = 0
        
        H1_br_bool = any(some_path[a:b+1] > H1)
        H2_br_bool = any(some_path[a:b+1] < H2)
        
        # If there was an H1 or H2 breach, find the earliest index of this breach.
        # Otherwise this index is infinity.
        if H1_br_bool:
            h1bt = some_path[a:b+1] > H1
            H1_br_loc = min(np.where(h1bt)[0]) + a
        else:
            H1_br_loc = np.infty
        if H2_br_bool:
            h2bt = some_path[a:b+1] < H2
            H2_br_loc = min(np.where(h2bt)[0]) + a
        else:
            H2_br_loc = np.infty
        
        # We use only 1 underlying stock.
        # It's impossible to breach both barriers at the same time, unless H1=H2
        if H1_br_loc < H2_br_loc:
            # converted to call with strike S(a)
            g_payoff = max(path[H1_br_loc]-path[a],0)
        elif H2_br_loc < H1_br_loc:
            # converted to a put with strike S(a)
            g_payoff = max(path[a]- path[H2_br_loc],0)
        else:
            # This is an error. We return junk to force an error.
            g_payoff = "Issue in Garrison Function."
        return g_payoff
    
    for path in sp_matrix:
        # At time a, stock price must be between the barriers.
        if ((path[a] > H1) or (path[a] <H2)):
            h_list.append(0)
        # Between a and b, we can't exceed the barriers at all.
        # The 'any' function checks for this
        elif ( any(path[a:b+1] > H1) or any(path[a:b+1] < H2) ):
            g_payment = garrison_breach(path)
            h_list.append(g_payment)
        else:
            v_call = max(path[Tm]-path[b],0)
            v_put  = max(path[b]-path[Tm],0)
            v_payment = max(v_call,v_put)
            h_list.append(v_payment)
    return h_list

For simplicity we will take

  • the end of the Waiting period, $a$, to be after $20\%$ of the option's life;
  • the end of the Garrison area, $b$, to be after $80\%$ of the option's life;
  • H1 and H2 to be $\pm \sigma \cdot T$ from our starting price $S_0$.

The last point makes sense, since (usually) $\sigma$ is anualized volatility, and $T$ is given in years. In $1$ year, we'd asusme the stock could rise or fall by $\sigma$. Therefore, this is a safe range or the barriers, that reduces the chance of breaching the Garrison to a repestable level.

IV. Scenario Analysis

Given the struture of the product, there are many scenarios, like edge-cases, we can take a look at. Some of these scenarios raise some questions about the structure of the product that haven't been discussed yet.

  • What if $H_1 = H_2$? (Should this be allowed?)
  • What if $b=a$ or $a=b$ ?
    $\circ$ A shorter/longer waiting/chooser period
    $\circ$ No Garrison. (Should this be allowed?)
  • What are the best and worst case scenarios?

Best Case Scenarios

Product Buyer

  1. Stock price rises steeply after being below $H_1$ until time $a$.
    $\bullet$ best case strike price is $H_2$.
    $\bullet$ payoff $max(S_T - H_2,0)$, where $H_2 = S_a$

  2. Stock price drops steeply after being above $H_2$ until time $a$
    $\bullet$ best case strike price is $H_1$.
    $\bullet$ payoff $max(H_1 - S_T,0)$, where $H_1 = S_a$

    $\circ$ In the case of a put, the max payoff is $H_1$ since the stock can only go to zero.

  3. Exact same stock price movement as Case-1 or Case-2, but product exits the Garrison
    $\bullet$ to exit the Garrison, the price must between $H_1$ and $H_2$ at $t_b$
    $\bullet$ The Vanilla chosen would maximize payoff.

    $\circ$ Best case scenario: (same payoff as Case-1) $S_b=K=H_2$ & stock price rises to $\infty$
    $\circ$ Best case scenario: (same payoff as Case-2) $S_b=K=H_1$ & stock price drops to $0$

  4. In our Example
    $\bullet$ $S_0, K, r, q, vol, T$ = 100, 100, 0.05, 0, 0.3, 1
    $\bullet$ $H_1 = S_0\cdot(1+vol)=130$
    $\bullet$ $H_2 = S_0\cdot(1-vol)=70$

    $\circ$ Best payoff on Case-1 is unlimited: $max(S_T-70,0)$
    $\circ$ Best payoff on Case-2 is 130: $max(H_1-S_T,0) = max(130-0,0) = 130$
    $\circ$ Case-3 mirrors these.

In each case, the option holder would also have spent money to purchase the option. From above, this would be an additional expense of $14.04$ on average (for a product with a 1 year expiry).

If the structured product were sold, then the seller would face potential losses of $130-14.04=115.96$ in Case-2, and unlimited losses in Case-1. These are in fact Seller "Worst" case scenarioa, but worth mentioning here since they are counterparts to the above.

Product Seller

In this case, there's no payoff only profit. As such, we'd like the option to either cease to exist, or expire worthless.

  1. Same as Buyer Worst Case-1 below. Profit of the premium.
  2. Same as Buyer Worst Case-2 below. Profit of the premium.
  3. Same as Buyer Worst Case-3 below. Profit of the premium.
  4. Stock price drops to or stays below $S_a$ until time $T$, after first exceeding $H_1$
    $\bullet$ This is one best case, for a seller, should the option enter the Garrison.
    $\bullet$ The breach of $H_1$ between $t_a$ and $t_b$ converts the product to a call with strike $S_a$
    $\bullet$ payoff (long position) is zero since $S_t$ is $\le$ $S_a$ at expiry
    $\circ$ profit to the Seller is the option premium.
  5. Stock price rises to or stays above $S_a$ until time $T$, after first breaching barrier $H_2$
    $\bullet$ This is the a sellers other best case, should the option enter the Garrison.
    $\bullet$ The breach of $H_2$ between $t_a$ and $t_b$ converts the product to a put with strike $S_a$
    $\bullet$ payoff (long position) is zero since $S_t$ is $\ge$ $S_a$ at expiry
    $\circ$ profit to the Seller is the option premium.

Worst Case Scenarios

Product Buyer

  1. Stock price exceeds $H_1$ or is less than $H_2$ at time $a$
    $\bullet$ payoff is zero, since the product ceases to exist

  2. Stock price at time $t=T$ equals stock price at time $t=b$
    $\bullet$ This is the worst case scenario if the product exits the Garrison.
    $\bullet$ payoff is zero since $S_b$ is the strike price for the chosen Vanilla Call and Vanilla Put

  3. Stock price at time $t=T$ equals stock price at time $t=a$
    $\bullet$ This is the worst case scenario if the price ever breaks through either barrier after the waiting period.
    $\bullet$ payoff is zero since $S_a$ is the strike price for the converted Call or Put

$\circ$ Buyer Summary: There are various ways to get a $0$ payoff. In terms of profit, there is a loss of the product premium. Thus the maximum loss for a buyer of the product is, just the price of the product - for example, $14.04$.

Product Seller

In addition to the complementary scenarios mentioned in Best Case, the seller of this product can lose in a few more ways. We'll list all of them.

  1. Same as Buyer Best Case-1. Potential for unlimited losses.
  2. Same as Buyer Best Case-2.
    $\bullet$ worst case loss of $H_1 - option$ $premium$
    $\bullet$ $130-14.04=115.96$ in our example
  3. Same as Buyer Best Case-3
    $\bullet$ Unlimited loss should stock price go to $\infty$
      $\circ$ holder would select Vanilla Call, and exercise at at strike as low as $K=S_b=H_2$
      $\circ$ As the seller, we'd have to buy on the market to deliver the stock
    $\bullet$ Maximum loss of $S_b - S_T$ should stock price go to $0$
      $\circ$ holder would select Vanilla Put, and exercise at at strike as high as $K=S_b=H_1$
      $\circ$ As the seller, we'd have to buy a (near) worthless stock from the exercisee of the put

$\circ$ Seller Summary: Overall, the Seller has less advantageous positions using this product. This is because, in various cases, the product converts in a way that is more useful to the buyer. Note that in the Seller Worst Case-3, if the price actually was zero, the buyer could likely not "buy" a $0$-valued stock to sell to the seller, but we understand the premise.

V. Sensitivity Analysis

Aside - Computational Problems:
    The array object in the NumPy package was not designed for operations like "appending". Rather, it was designed/optimized for math operations. Unlike lists and other similar objects in Python, the NumPy array functions `concatenate, hstack, vstack` create a copy of the object being worked with. In our simulation above, this meant that while we tried to avoid the issues of having a double loop, it was still wildly inefficient using memory.
    For Sensitivity Analysis, we'll rebuild the _simulator_ using just lists. Then we will convert the final output to a NumPy array, so that we can still use the payoff function above. This should speed up the creation/storing of multiple paths significantly.

# original version run via terminal

import numpy as np
# define the variables
_iv = {'s0':100,'K':100,'r':0.05,'q':0,'vol':0.3,'T':1}
_load= dict(N=30,M=60)

# define barriers and cut-off points
def limits(iva,ldb):
    a_calc = int(0.2*ldb['N'])
    b_calc = int(0.8*ldb['N'])
    h1_calc = iva['s0']*(1+iva['vol'])
    h2_calc = max(iva['s0']*(1-iva['vol']),0) # since stock price can't be negative
    lim = {'a': a_calc,'b': b_calc,'H1': h1_calc,'H2': h2_calc}
    return lim

# where to write data
# we can add a newline for each sensitivity test
discountfile = 'sense_tests'

# where to put each path
list_of_paths = []

# define geo brownian mtn
def g_stock_path(s0,K,r,q,vol,T,N,M):
    """Generates N values of stock price from time 0 to T.
    Stock price follow geometric brownian motion.
    Uses lists, instead of numpy arrays"""
    dt=T/N
    s = [s0]
    temp1 = ((r - q) - (0.5 * vol*vol))*dt
    temp2 = vol * np.sqrt(dt)
    for step in range(N):
        next_val = s[step] * np.exp(temp1 + (temp2 * np.random.randn()))
        s.append(next_val)
    return s

# generate multiple paths and put them above
def g_sims(s0,K,r,q,vol,T,N,M):
    for sims in range(M):
        new_path = g_stock_path(s0,K,r,q,vol,T,N,M)
        list_of_paths.append(new_path)
        if ((sims+1)%5000==0):
            print(sims+1)

# define the payoffs
def payoff_calc(sp_matrix,check_dict):
    h_list = []
    # Garrison entry and exit times
    a,b = check_dict['a'], check_dict['b']
    # Full time to maturity
    Tm = len(sp_matrix[0])-1
    # High and Low price barriers
    H1,H2 = check_dict['H1'], check_dict['H2']
    
    def garrison_breach(some_path):
        """ find the payoff in the case the product was converted due to 
        exceeding any barrier, at all, between times a and b."""

        some_path = np.array(some_path)
        
        g_payoff = 0
        
        H1_br_bool = any(some_path[a:b+1] > H1)
        H2_br_bool = any(some_path[a:b+1] < H2)
        
        # If there was an H1 or H2 breach, find the earliest index of this breach.
        # Otherwise this index is infinity.
        if H1_br_bool:
            h1bt = some_path[a:b+1] > H1
            H1_br_loc = min(np.where(h1bt)[0]) + a
        else:
            H1_br_loc = np.infty
        if H2_br_bool:
            h2bt = some_path[a:b+1] < H2
            H2_br_loc = min(np.where(h2bt)[0]) + a
        else:
            H2_br_loc = np.infty
        
        # We use only 1 underlying stock.
        # It's impossible to breach both barriers at the same time, unless H1=H2
        if H1_br_loc < H2_br_loc:
            # converted to call with strike S(a)
            #g_payoff = max(path[H1_br_loc]-path[a],0)
            g_payoff = max(path[H1_br_loc]-path[a],0)
        elif H2_br_loc < H1_br_loc:
            # converted to a put with strike S(a)
            g_payoff = max(path[a]- path[H2_br_loc],0)
        else:
            # This is an error. We return junk to force an error.
            g_payoff = "Issue in Garrison Function."
        return g_payoff
    
    for path in sp_matrix:
        path = np.array(path)
        # At time a, stock price must be between the barriers.
        if ((path[a] > H1) or (path[a] <H2)):
            h_list.append(0)
        # Between a and b, we can't exceed the barriers at all.
        # The 'any' function checks for this
        elif ( any(path[a:b+1] > H1) or any(path[a:b+1] < H2) ):
            g_payment = garrison_breach(path)
            h_list.append(g_payment)
        else:
            v_call = max(path[Tm]-path[b],0)
            v_put  = max(path[b]-path[Tm],0)
            v_payment = max(v_call,v_put)
            h_list.append(v_payment)
    return h_list

# the_payoffs = payoff_calc(list_of_paths,limits)
def discounted_payoffs(pyf):
    r,T = _iv['r'],_iv['T']
    return np.average(pyf) * np.exp(-r*T)

def store_trial(s0,K,r,q,vol,T,N,M,lim_replace=''):
    global list_of_paths
    list_of_paths = []
    
    
    # reassign default vars with what's passed in
    _iv['s0'],_iv['K'],_iv['r'],_iv['q'],_iv['vol'],_iv['T'] = s0,K,r,q,vol,T
    _load['N'],_load['M'] = N,M
    
    
    
    g_sims(_iv['s0'],_iv['K'],_iv['r'],_iv['q'],_iv['vol'],_iv['T'],_load['N'],_load['M'])
    if not lim_replace:
        limz = limits(_iv,_load)
    else:
        limz = lim_replace

    payoffs = payoff_calc(list_of_paths,limz)
    dsc_payoffs = discounted_payoffs(payoffs)

    out_put = open(discountfile,'a')
    myline = str(dsc_payoffs) + ',' + str(_load['N']) + ',' + str(_load['M'])

    abh1h2 = [str(v) for v in list(limz.values())]
    abh1h2_joined = ','.join(abh1h2)
    
    myline +=','
    myline += abh1h2_joined
    
    baseparams ="{},{},{},{},{},{}".format(_iv['s0'],_iv['K'],_iv['r'],
                                           _iv['q'],_iv['vol'],_iv['T'])
    
    myline +=','
    myline += baseparams
    out_put.write(myline)
    out_put.write('\n')
    out_put.close()

    k. Garrison Width Special Case

    $H_1 = H_2$

    This is a near impossible case. At time $t_a$, if $H_1=H_2$ then

    • $S_a$ will either be above both, below both or equal to both
      • If $S_a$ is not equal to both, then at time $t_a$ it cannot be both below $H_1$, and above $H_2$.
      • Therefore the product ceases to exist.
